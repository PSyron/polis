# Release notes: 0.1.0

Date: 2026-07-20

## Scope

This release publishes the first stable milestone of Polis and confirms:

- privacy-safe offline behavior,
- package and dependency audit alignment,
- clean-distribution build/install validation,
- reproducible release artifact checks.

## Highlights

- Added acceptance coverage for realistic Polish correction behavior in JSON/XML
  fixtures and end-to-end test flow.
- Added explicit release-audit evidence for network attempts, secrets, and artifact
  contents.
- Added M4 distribution validation commands for build, metadata checks, and clean
  installation smoke tests.
- Added source-policy `1.1`, which automatically applies comma insertions from
  the five explicitly qualified Polish LanguageTool rule IDs for sentences.
- Added optional, reviewable contextual inflection suggestions for sentences
  through the pinned local LanguageTool stdio module.

## Known limitations

- The current local backend behavior is conservative and rule-focused.
- Broader stylistic rewriting remains out of scope.
- No document-container adapters (DOCX/ODT/RTF) are included in this release.
