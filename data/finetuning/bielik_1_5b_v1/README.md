# Polis Bielik 1.5B correction dataset v1

This bundle contains project-authored Polish correction examples for a narrow,
offline Bielik 1.5B adapter. It is independent of the evaluation corpus and may
be used for fine-tuning.

## Contents

- `train.jsonl`: 1,200 records, 300 per category;
- `validation.jsonl`: 240 records, 60 per category;
- `manifest.json`: deterministic counts, review states, coverage, and file
  hashes;
- `LICENSE`: CC0-1.0 dedication for this bundle.

The four categories are inflection, syntax, punctuation, and protected
no-change examples. Inflection examples select a finite candidate. Other
examples return corrected text. Every row stores structured messages and a
rendered ChatML audit field compatible with the official Bielik 1.5B v3
tokenizer template.

## Creation and review method

All source sentences and corrections are authored for Polis. No source text or
gold answer was generated by a language model. The generator combines explicit
name paradigms, agreement transformations, punctuation transformations, and
protected-negative templates. Those transformation registries were reviewed as
linguistic units; records retain the transformation and template identifiers.

Regenerate the bundle from the repository root:

```bash
uv run python scripts/generate_finetuning_dataset.py
```

Then validate it against the frozen evaluation corpus:

```bash
uv run pytest tests/test_finetuning_dataset.py
```

The validator rejects malformed prompts, unsupported categories, duplicates,
no-op positives, unsafe rewrites, split overlap, and text/template/entity or
expected-output overlap with corpus v3. The train and validation lexicons and
template registries are separate.

## Limitations

The data is synthetic and deliberately narrow. It is not representative of all
Polish domains or error types. Validation measures fit to these controlled
transformations; corpus v3 remains the independent product-quality evaluation.
Structured `messages` should be passed through the selected tokenizer's
`apply_chat_template`; the stored `chatml` field is intended for audits and
reproducibility.
